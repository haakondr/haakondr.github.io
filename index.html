Husbanken - Grunnlån esøknad

August 2017 - Mai 2018

Rolle: Tech lead, utvikler

Grunnlån esøknad er en webbasert søknadsapplikasjon som integrerer mot aktuelle registre og bistår søkeren med utfylling av søknader om grunnlån eller tilskudd fra Husbanken.

Applikasjonen er en Single Page Application skrevet i Angular som består av et ganske komplekst sett med visningslogikk basert på brukerroller og valg man gjør underveis i søknaden, samt data fra eksterne registre (matrikkelen, enhetsregisteret, osv.)

Prosjektet banet også vei for å etablere en felles kundetjeneste på tvers av applikasjoner og fagområder i Husbanken.

Min rolle var i hovedsak å skisse opp og koordinere frontend-arkitektur, samt utvikling i prosjektet. Deltok også i brukertester ute hos sluttbrukere.

Prosjektet består av fem utviklere, og rundt åtte fag/ux/devops ressurser.

Teknologi:

Angular, TypeScript, Redux, Java, Spring boot, JAX-RS, JWT, Hibernate, JPA, Apache Kafka, RabbitMQ, Sql Server, Docker, Kubernetes, Openshift, Jenkins, Wiremock, Docker Compose, Splunk, Selenium, Cucumber

Husbanken - Startlån Saksbehandling

Mars 2015 - September 2017

Rolle: Tech lead, utvikler

Startlån saksbehandling tar imot søknader fra eSøknad Startlån. Applikasjonen bistår i saksbehandlingen, og tilbyr funksjonalitet som simulering av fremtidssituasjoner, arkivering, interaksjon med søkere, mm. for kommunale saksbehandlere.

Min rolle gikk primært ut på å skissere opp overordnet arkitektur, fasilitering av API-design, parprogrammering, pull requests osv. Mange prosessmessige grep ble gjort for å kunne levere hyppige endringer på et system som er i bruk. Bistod også med intervjuer i forbindelse med anskaffelser.

Prosjektet bestod av tolv utviklere, og rundt ti fag/ux/test/devops ressurser.

Husbanken - Startlån eSøknad

Jan 2014 - Juni 2017

Rolle: Utvikler (frontend, backend)

eSøknad Startlån ble lansert 4. mai 2016. Ved å kunne sende inn søknad elektronisk sparer både søkere og saksbehandlere mye tid. Applikasjonen tar imot ca 1000 innsendte søknader i måneden. Prosjektet vant i 2016 prisen for Årets Digitale Tjeneste [1].

Hadde en bred rolle som utvikler som omfattet alt fra frontend og backend utvikling til testautomatisering og konseptutvikling. Deltok også på diverse workshops med fagressurser og interaksjonsdesignere der det var fokus på konsept/flyt/utforming av løsningen.

Prosjektet hadde et tydelig fokus på automatiserte ende-til-ende browsertester vha. cucumber/protractor/selenium. Automatiserte tester kombinert med docker gjorde det mulig å kjøre fullstendige ende-til-ende tester lokalt på hver enkelt utviklerpc. Samtidig kjørte de samme testene i continuous integrastion med automatiske releaser til testmiljøet.

Gjennom hyppige brukertester jobbet teamet iterativt frem til en endelig løsning. Det var derfor viktig å kunne respondere raskt på endringer. Her ble det lagt ned mye arbeid med en komponentbasert arkitektur, med små testbare komponenter i AngularJS som lett kunne flyttes rundt i skjermbildene.

Hadde ansvar for flere nøkkelfunksjoner, blant annet:

  • Klientsidelogging - Alle JavaScript feil logges med stacktrace samt et utdrag av brukerinteraksjoner som førte til feilen
  • Altinn-integrasjon - henter ut selvangivelsedata og dokumenter basert på innlogget sesjon i Altinn med cross origin requests mot altinn sitt API

[1]: https://kvalitet.difi.no/artikkel/2016/06/prisen-arets-digitale-tjeneste-til-husbanken

Teknologi:

AngularJS, JavaScript, MongoDB, Java, Docker, Selenium, NodeJS, HTML5, CSS3

NTNU - Graph-based Natural Language Processing

Jan 2012 - Desember 2012

Masteroppgave

Oppgaven gikk ut på å representere tekst som graf-strukturer og bruke strukturelle likheter som et similaritetsmål. En graph edit distance algoritme ble implementert som kalkulerte den strukturelle distansen mellom to setnigner. Algoritmen ble brukt som distansemål i en kNN (k-Nearest Neighbours) klassifisering.

Tilnærmingen ble målt mot andre algoritmer til å detektere plagiat i et ferdigdefinert datasett i forbindelse med CLEF 2011 konferansen.

Link til oppgaven: https://brage.bibsys.no/xmlui/handle/11250/253133

Teknologi:

Java, MaltParser, Stanford NLP, Python, Apache Lucene
